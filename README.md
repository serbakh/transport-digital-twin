# Digital Twin of a Transport Forwarding Process for Synthetic Data Generation

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.21363806.svg)](https://doi.org/10.5281/zenodo.21363806) 
Програмний код, синтетичні набори даних і результати експериментів архівовано в Zenodo: DOI 10.5281/zenodo.21363806.

## Опис проєкту
Проєкт реалізує цифровий двійник транспортно-експедиторського процесу на основі дискретно-подієвого моделювання та демонструє можливість використання синтетичних даних для навчання моделей машинного навчання.

Цифровий двійник відтворює повний цикл доставки вантажу від складу до борта судна, враховуючи обмежені виробничі ресурси, випадкові коливання тривалості технологічних операцій, виробничі перерви та зовнішні фактори.

На основі сформованих синтетичних даних будуються моделі машинного навчання, які використовуються для прогнозування продуктивності транспортно-експедиторського процесу та підтримки прийняття управлінських рішень.

# Основні модулі

## digital_twin.py

Модуль цифрового двійника.

Виконує дискретно-подієве моделювання транспортно-експедиторського процесу та формує синтетичні набори даних.

Результати:

- scenario_summary.csv
- trips_all_scenarios.csv

---

## train_model.py

Навчання моделей машинного навчання.

Виконується:

- прогнозування добового тоннажу;
- прогнозування вузького місця системи;
- оцінювання моделей (включно з 5-fold крос-валідацією);
- порівняння з базовою лінійною моделлю (Linear Regression) для обґрунтування вибору Random Forest;
- аналіз важливості ознак;
- розрахунок кривої навчання (залежність якості моделей від обсягу даних);
- збереження навчених моделей.

Результати:

- моделі Random Forest;
- метрики якості;
- learning curve;
- feature importance.

---

## recommend_resources.py

Модуль підтримки прийняття управлінських рішень.

Використовує навчені моделі машинного навчання для визначення мінімальної конфігурації виробничих ресурсів, необхідної для виконання договірного плану.

Враховуються:

- кількість самоскидів;
- кількість мехлопат;
- кількість точок розвантаження;
- зовнішнє навантаження вагового комплексу;
- дорожні затримки.

---

## plot_results.py

Формує графічні матеріали для аналізу результатів експериментів.

Будуються:

- продуктивність системи;
- часи очікування;
- завантаженість ресурсів;
- вплив зовнішніх факторів;
- вплив обсягу синтетичних даних на якість моделей (learning curve).

# Послідовність запуску

## 1. Генерація синтетичних даних

```bash
python src/simulation/digital_twin.py
```

Будуть сформовані:

```
data/synthetic/
```

---

## 2. Навчання моделей машинного навчання

```bash
python src/ml/train_model.py
```

Будуть сформовані:

```
data/ml/
models/
```

---

## 3. Формування рекомендацій

```bash
python src/ml/recommend_resources.py
```

Буде сформовано:

```
data/ml/resource_recommendations.csv
```

---

## 4. Побудова графіків

```bash
python src/analysis/plot_results.py
```

Потребує наявності `data/ml/ml_learning_curve.csv` (формується на кроці 2).

Будуть сформовані:

```
data/plots/       (5 графіків)
data/ml/          (2 графіки learning curve)
```

# Наукове призначення

Проєкт створено як програмну реалізацію дослідження, присвяченого використанню цифрового двійника транспортно-експедиторського процесу як джерела причинно-узгоджених синтетичних даних для навчання моделей машинного навчання.

Основною особливістю запропонованого підходу є те, що синтетичні дані формуються не випадковою генерацією, а в результаті моделювання реального виробничого процесу зі збереженням причинно-наслідкових зв'язків між технологічними операціями, виробничими ресурсами та показниками продуктивності.
